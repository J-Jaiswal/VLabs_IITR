Finding location of an earthquake

UI Controls

  • Stations — how many receivers to place when you click Generate. Use at least 4. It’s better if stations surround the source instead of sitting in a line.
  • Event (X, Y, Z) — the “real” earthquake we use to make fake (synthetic) picks. >
  • vtrue — velocity used to create the synthetic data.
  • vstart — velocity assumed by the solver and the misfit map.
  • σ — noise level on the picks (seconds).
  • Outlier — add one bad pick (mild ±3σ or strong ±6σ).
  • λ — Tikhonov damping. Bigger λ makes steps safer/smoother but slower, and can add small bias.
  • Generate / Run / Reset — main buttons. Generate places stations, Run solves for [x,y,z,t0,v], Reset clears and loads defaults.

Things to test

  1. Baseline: Good station spread, vtrue=vstart, small σ. You should see a centered low-error region and the RMS (average error) drops quickly.
  2. Velocity bias: Keep stations the same but make vstart ≠ vtrue. The low-error area moves, so the location shifts (bias) even if RMS looks small.
  3. Noise test: Increase σ to ~0.2–0.3 s. RMS finishes higher, the estimate wiggles more, and uncertainty grows.
  4. Outlier test: Add a mild/strong outlier. Turn up λ until the solution becomes steady again.
  5. Geometry test: Regenerate until stations are almost in a line or clumped. Expect a stretched uncertainty shape and slower, less reliable convergence.

Step-by-step workflow

1) Place stations

  1. Choose Stations and click Generate to place a random network.
  2. If needed, click again until stations surround the expected source area (good spread).

2) Set the “true Event" and the starting guess

  1. Set the true Event (X, Y, Z), vtrue, and noise σ. You can also add an Outlier.
  2. Set vstart for the solver and misfit map (start with vstart=vtrue).

3) Run the inversion and read the plots

  1. Click Run. Each iteration the solver:
    • Predicts arrival times from the current guess [x_s,y_s,z_s,t_0,v].
    • Computes residuals (observed − predicted) and updates RMS.
    • Builds a sensitivity matrix (Jacobian) and takes a damped Gauss–Newton step.
    • Updates the model until changes are tiny or we hit the max iterations.
  2. Map view: triangles = stations (amber ring = outlier station if any); big star = true source; small star = current estimate. Watch the small star move toward the big one.
  3. Misfit map: a 2D scan that shows where errors are smallest. Bright area = lower mean-square error. If it’s not over the true star, your vstart is probably off.
  4. Convergence: RMS vs iteration should go down and settle near the noise level. If it bounces or rises, increase λ or use a better station spread/start velocity.

4) Tune and repeat

  1. If the updates look jumpy or the estimate keeps drifting, increase λ and run again.
  2. If the low-error region looks stretched or not centered on the true star, regenerate stations so they surround the source better.
  3. Try vstart ≠ vtrue to see how using the wrong velocity shifts (biases) the location.
