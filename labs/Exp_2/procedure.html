Finding location of an earthquake

Learning goals

  • See how station layout, noise, and assumed velocity change accuracy and stability.
  • Read the misfit map, convergence (RMS) curve, and the 1-σ uncertainty ellipse.
  • Use damping λ to keep the solution stable when geometry/noise is bad.

Controls

  • # Stations — how many receivers you place. Use ≥ 4. A wide spread around the source beats just adding more.
  • Event (X, Y, Z) — the “true” earthquake you set for the simulator. Used to generate the synthetic arrival times.
  • vtrue — velocity for generating data. Represents the real Earth in this demo.
  • vstart — velocity used by the inversion and the misfit scan. If wrong, the estimate can be biased even if RMS looks small.
  • σ — Gaussian noise level on picks (seconds). Higher σ ⇒ bigger ellipse and slower/messier convergence.
  • Outlier — optionally add one bad pick. Good for testing robustness; can destabilize the solver.
  • λ — damping strength. Larger λ stabilizes updates but can slow movement toward the answer.
  • Add / Drag / Remove — network design tools. Controls resolution and the condition number of the problem.

Experiments to perform

  1. Baseline: good geometry, v_{true}=v_{start}, small σ. Expect a centered bowl, small ellipse, quick convergence.
  2. Velocity bias: fix geometry; set v_{start} ≠ v_{true}. The misfit bowl shifts; estimate is biased.
  3. Noise test: raise σ to 0.2–0.3 s. Ellipse grows; RMS flattens.
  4. Outlier test: add a mild/strong outlier. Increase λ to stabilize.
  5. Geometry test: line up or cluster stations. cond(G) rises; ellipse elongates.

Step-by-step workflow

1) Set up the stations

  1. Place 6–8 stations and drag them to surround the expected source (avoid a straight line).
  2. Aim to reduce cond(G) (better azimuthal coverage ⇒ more stable inversion).

2) Choose the “true Earth” and a starting guess

  1. Set Event (X, Y, Z), vtrue, and noise σ.
  2. Pick vstart for inversion; start with v_{start}=v_{true} to build intuition.

3) Run and read the plots

  1. Click Run. Each iteration:
    • Predicts the arrival times for all stations from the current guess of the source location, origin time, and velocity.
    • Compares predictions with the observed picks to get the mismatch for each station and an overall RMS (average error).
    • Figures out sensitivities—which parameters (x, y, z, origin time, velocity) would reduce the error the most if nudged.
    • Updates the model gently using damping so changes are stable and don’t overreact to noise/outliers.
    • Repeats until the change is tiny or a set number of steps is reached.
    • Good signs: RMS drops most steps, the blue estimate settles, and updates get smaller.
    • Warnings: RMS stalls or rises, the estimate jumps around, or the problem looks ill-conditioned—try spreading stations out or increasing damping.
  2. Misfit map: The app scans many possible epicenters using the chosen start velocity. For each point it picks the best origin time and colors the map by how well it fits.

    Look for a bright “bowl.” Its center is the best epicenter for that fixed velocity. If the bowl sits away from the true (red) point, your start velocity is likely off. A long, skinny bowl means poor station geometry.

  3. Map: Red is the true source, blue is the current estimate, and the purple dashed ellipse shows the model’s horizontal uncertainty.

    A bigger ellipse means lower confidence. Its orientation points along the direction your network constrains poorly. You can shrink it by adding/spreading stations to improve coverage.

  4. Convergence: The RMS vs iteration plot should generally go down.

    If it stalls: increase damping, move/add stations for better coverage, remove the outlier/noise, or choose a better start velocity.
